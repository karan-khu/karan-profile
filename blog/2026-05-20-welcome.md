## ทำไมต้องมี activity log

จากที่ทำงานสายเทคมาสักพัก ผมพบว่ากิจกรรมที่ทำให้เรียนรู้เยอะที่สุดมัก
**ไม่ใช่งานประจำ** แต่เป็นช่วงเวลาที่ได้:

- เตรียมหัวข้อไป share ในทีม
- ตอบคำถามตอน workshop
- ฟัง talk แล้วเอาไอเดียกลับมาลอง

หน้านี้เลยตั้งใจให้เป็นที่บันทึกกิจกรรมพวกนั้น ทั้งเพื่อ:

1. ให้ตัวเองในอนาคตย้อนกลับมาดูได้
2. ให้คนที่สนใจเห็นว่าผมทำอะไรอยู่
3. แชร์เนื้อหา / slide ที่ใช้ในงาน

## วิธีอ่าน

โพสต์ใหม่จะอยู่บนสุด แต่ละโพสต์มี tag ระบุประเภท เช่น
`meeting`, `teaching`, `training`, `talk` คลิก tag เพื่อกรองได้

## วิธีเพิ่มโพสต์ใหม่

สร้างไฟล์ Markdown ใน `blog/` ตามรูปแบบ `YYYY-MM-DD-slug.md` แล้วใส่
frontmatter ด้านบน เช่น

```md
---
slug: meeting-2026-06-share-tech
title: Share tech — เรื่องที่อยากเล่า
authors: [karan]
tags: [meeting]
---

เนื้อหา...
```

push ขึ้น GitHub แล้วเว็บจะ rebuild ให้อัตโนมัติ (ถ้าเปิด GitHub Actions
ตาม workflow ที่เตรียมไว้ใน repo)
